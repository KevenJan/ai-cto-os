# AI CTO OS

> AI驱动的软件研发操作系统（从需求到上线的全流程自动化体系）

---

## 🚀 它是什么？

AI CTO OS 不是一个知识库，而是一个：

> 将“需求 → PRD → 原型 → 代码 → 部署”标准化的 AI 软件工厂

---

## 🧠 核心能力

- 📋 AI自动需求分析
- 📄 自动生成PRD
- 🎨 自动生成原型设计
- 🏗 自动系统设计（数据库 / API）
- 💻 自动代码生成（Java / Vue / 小程序）
- 🧪 自动测试生成
- 🚀 自动部署方案

---

## ⚙️ 使用方式

### Step 1：写一句需求

例如：

> 我要做一个修理厂会员系统

---

### Step 2：进入 workflows

选择对应流程：

- 需求分析
- PRD生成
- 系统设计
- 开发
- 测试
- 部署

---

### Step 3：使用 prompts

调用对应AI角色：

- 产品经理
- 架构师
- Java开发
- 测试工程师

---

### Step 4：从 examples 复用

已有项目直接复制修改：

- repair-shop
- car-wash
- gaokao-platform

---

## 🏗 目录结构说明

- prompts → AI角色能力库
- templates → 标准文档模板
- examples → 可复用项目案例
- workflows → 软件生产流程
- agents → AI角色系统（未来扩展）
- cursor-rules → Cursor开发规范
- claude-rules → Claude Code规范
